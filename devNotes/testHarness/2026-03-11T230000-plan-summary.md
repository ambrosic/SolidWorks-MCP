# Test Harness v1 - Development Log

## 2026-03-11 - Plan Finalized

### Goal
Build a Layer 2 AI behavior eval harness for the SolidWorks MCP server. Feeds CAD tasks to an LLM (via Ollama/OpenAI-compatible API), lets it make tool calls against live SolidWorks, records everything, and grades results across 5 dimensions.

### Architecture Summary
- `testHarness/models.py` - Core dataclasses (ToolCall, EvalSession, TaskSpec, GradeResult)
- `testHarness/middleware.py` - InstrumentedServer wrapping SolidWorksMCPServer._route_tool()
- `testHarness/llm/` - Provider-agnostic LLM adapter (OpenAI-compatible API)
- `testHarness/runner.py` - Tool-calling conversation loop
- `testHarness/graders/` - 6 graders (coverage, order, efficiency, error_recovery, geometry, judge)
- `testHarness/tasks/` - YAML task definitions
- `testHarness/run_eval.py` - CLI entry point

### Grading Weights
- Geometry: 30% (ground truth - does the final shape match?)
- Coverage: 25% (were required tools called?)
- Order: 15% (sensible sequence?)
- Efficiency: 15% (minimal flailing?)
- Error Recovery: 15% (handled failures well?)

### Key Decisions
- Uses `requests.post` to OpenAI-compatible endpoint, not the `openai` package
- Ollama as primary LLM provider, system is provider-agnostic
- Git commit hash + branch included in every result for correlation
- Existing test.py and server.py untouched
- Results in JSON format, timestamped, gitignored

### Implementation Order
Phase 0: Dev notes setup (this file)
Phase 1: Foundation (models, middleware)
Phase 2: Graders (all 6)
Phase 3: Task definitions (3 starter YAMLs)
Phase 4: LLM integration (adapter layer)
Phase 5: Runner + CLI
Phase 6: Polish (judge, config, more tasks)
